# DnD Game — дизайн-документ

> Рабочее название: **DnD Game** (финальное — на M5). Статус: **v4** от 2026-07-09.
> Терминология — в [CONTEXT.md](../CONTEXT.md), необратимые решения — в [docs/adr/](adr/).
> Игра делается **для публичного релиза**.

## 1. Концепция

Сингловая игра-«квест-комната» в духе настольного DnD, где гейммастер — нейросеть.
ГМ придумывает сценарий под персонажа игрока, строит карту, предметы, NPC, механики
и графику, определяет условие победы — и (в полной версии) живёт в игре: реагирует
на действия, интерпретирует безумные Заявки, меняет мир. **Цель: ИИ-ГМ как
человек-ГМ** — характер, юмор, логика реального мира, эмоциональный интеллект,
главная ценность — сама игра и её интересность.

**Референсы:** Scribblenauts · TADC/Кейн · Stanley Parable · NetHack/Stoneshard
(модель времени) · Mantella (распространение).

**Два столпа поверх всего:**
- **DnD без стола** (ADR-0008): кубики, карточки, фигурки и усталость ГМа — костыли
  настольного формата; их заменяет симуляция. Игрок видит только мир и последствия.
- **Статичная играбельность** (ADR-0009): сгенерированный уровень полностью проходим
  без единого рантайм-вызова LLM. Живой ГМ — слой поверх надёжной игры, а не её
  условие.

**Ключевая петля:**
1. Игрок создаёт персонажа классически — раса, класс, скриптованные статы + немного
   свободных поинтов (Лист видим: невидимы броски, а не состояние) + лор-поле и
   особые Черты через Валидатора против имбы → ГМ с максимальным эффортом генерирует
   уровень целиком (сценарий, карта, NPC, Механики, Скриптовые триггеры, Эскалации,
   предикат победы).
2. Игра пошаговая (ADR-0007): вне боя Плавные ходы; в бою стоп-время, одно действие
   за Ход. Базовые действия — мгновенно в движке: нажал → бросок (невидимый) →
   урон → анимация. ГМ не комментирует каждый вздох.
3. Заявка (фаза Живого ГМ): игрок может сказать ГМу что угодно — ГМ оценивает
   реалистичность, назначает цену в ходах, переводит в Проверки и Намерения.
4. ГМ отвечает на каждую Заявку без фильтров, но у него есть Эмоциональное
   состояние: он не обязан угождать, может съязвить, а намеренного саботажника —
   Изгнать со стола. Софтлок запрещён; Изгнание — санкционированная концовка.
5. Предикат победы выполнен → финал → следующая локация/сессия.

## 3. Принципы

1. **Статичная играбельность.** Сначала игра, которая работает без ИИ; живой ГМ — усилитель, не костыль (ADR-0009).
2. **Детерминированное — коду, творческое — LLM.** Всё, где LLM галлюцинирует или ненадёжна, закрывается не-ИИ инструментом (см. 4.6).
3. **Важность решается на генерации.** ГМ — автор всех секретов, он сам расставляет Триггеры и Эскалации. В рантайме никто «не смотрит стрим».
4. **Задержка — это фича, а лучшая задержка — нулевая.** Статичное ядро отвечает мгновенно; Заявки маскируются паузой рассказчика и Многоходовостью.
5. **Примитивы ≠ игровые действия.** Конечный алфавит, неограниченные комбинации; лестница «композиция → Библиотека → код» (ADR-0004).
6. **DnD без стола.** Случайность — детерминированные невидимые броски движка (ADR-0008).
7. **Экономный инференс — архитектурное требование.** Секрет, который не нашли, не стоит ни токена.
8. **Контекст не растёт со временем.** Всё персистентное — БД с top-k выборкой, не контекст (см. 4.7).
9. **Мозг — сменный модуль.** Модель = конфигурация (Адаптер LLM).

## 4. Архитектура

### 4.1 Два режима работы ИИ

```
ГЕНЕРАЦИЯ (дорого, медленно, один раз)          РАНТАЙМ
┌──────────────────────────────────┐   ┌─────────────────────────────┐
│ Режиссёр (макс. эффорт):         │   │ Фаза A «Статичное ядро»:    │
│  сценарий, карта, NPC, секреты,  │   │  движок + Рефлексы,         │
│  Скриптовые триггеры, Эскалации, │   │  0 вызовов LLM              │
│  предикат победы, заказ Механик  │   │                             │
│ Внутренний грилл: Логик/Валидатор│   │ Фаза B «Живой ГМ» (M4+):    │
│  гоняют сценарий по дырам        │   │  + Заявки игрока,           │
│ Инженер: компилирует все Механики│   │  + Пробуждающие триггеры,   │
│  и реакции триггеров заранее     │   │  + Эскалации будят ГМа,     │
│ Валидатор: графовая проходимость │   │  + эмоциональная Персона    │
└──────────────────────────────────┘   └─────────────────────────────┘
```

### 4.2 Роли (все — контексты одной-двух моделей, ADR-0003)

| Роль | Когда работает | Задача |
|---|---|---|
| **Режиссёр** | Генерация; рантайм — только фаза B (Заявки, Пробуждающие триггеры) | Сценарий, секреты, Триггеры, Эскалации, реплики, NPC, баланс. Только Намерения, никогда код |
| **Инженер** | Генерация (всё заранее); рантайм фазы B — редкие новые Намерения | Компиляция Намерений по лестнице; новое — в Библиотеку навсегда |
| **Логик** | Генерация (грилл сценария); фаза B | Правдоподобие; Мягкое вето (переопределяется Правилом крутости) |
| **Валидатор** | Генерация; фаза B перед применением изменений | Жёсткое вето: софтлок, Канон, безопасность кода. Плюс детерминированные проверки (4.5) |
| **Рефлексы** *(код, не LLM)* | Рантайм, каждый ход | Исполнение Скриптовых триггеров, счётчики, Эскалации, Температура стола |

### 4.3 Триггеры: скриптовые, пробуждающие, эскалации

Канонический пример (яма):
- **Генерация:** Режиссёр прячет яму и решает: падение — не повод для его выхода.
  Пишет Скриптовый триггер: «упал → стан 2 хода, анимация; после стана — базовое
  действие "вылезти" с проверкой ловкости/выносливости». Инженер компилирует это
  в Механику **ещё до старта игры**.
- **Рантайм (фаза A):** игрок упал → мгновенно: анимация, стан, новое действие
  в меню. Ноль LLM. Заявки поверх («копаю тоннель») — фаза B.
- **Эскалация:** Режиссёр на генерации задал: «10 провалов "вылезти" подряд →
  вмешаться». Счётчик Рефлексов дёргает Пробуждающий триггер → живой ГМ решает по
  контексту: заспавнить врагов, которые «его туда и поймали», или сжалиться:
  «Спустя час попыток ты замечаешь корни на стене. +5 к проверке. Не благодари».
- **Пробуждающий триггер** ставится сразу там, где ГМ сам счёл событие достойным
  своего выхода (вход в тронный зал, находка ключевого предмета).

### 4.4 Латентность по путям

| Путь | Фаза | Бюджет | Маскировка |
|---|---|---|---|
| Все базовые действия, бой, Проверки, Скриптовые триггеры | A | 0 мс | не нужна |
| Реплика на Пробуждающий триггер / Эскалацию | B | 1–4 с | асинхронно, «догоняет» между ходами; бой не ждёт нарратив |
| Заявка (единственный блокирующий путь — игрок сам ждёт вердикт) | B | 2–6 с | пауза рассказчика; ход противника |
| Заявка → новая Механика (кодогенерация) | B | 10–30 с | Многоходовость самого действия; реплика-филлер |
| Генерация уровня | — | минуты | экран создания персонажа, пролог-нарратив |

Мутации мира применяются строго между ходами.

### 4.5 Формальная проверяемость победы

Условие победы, придуманное Режиссёром, **компилируется в предикат** над Состоянием
мира; Валидатор проверяет достижимость детерминированно: связность карты (поиск
пути), существование цепочки «инструменты → действия → цель». LLM-суждение — поверх,
а не вместо. Нарративная обёртка цели — любая.

### 4.6 Детерминированные метрики вместо ИИ-суждений

Список задач, где LLM галлюцинирует/ненадёжна, и их не-ИИ инструменты
(живой список — пополняется по мере тестов):

| Задача | Риск LLM | Инструмент |
|---|---|---|
| Числа: HP, урон, инвентарь, деньги | арифметические ошибки | движок |
| Случайность | нечестный «рандом» | невидимые броски движка (ADR-0008) |
| Геометрия: путь, дальность, линия видимости | путает расстояния | сеточные алгоритмы движка |
| Проходимость уровня | оценка «на глаз» | предикат победы + графовые проверки (4.5) |
| Повторяющиеся неудачи, стагнация | не заметит / забудет | счётчики Рефлексов + Эскалации |
| Интересность игры | вкусовщина | **Температура стола**: формула из сигналов — провалы одной Проверки подряд, ходы без новых событий/зон/предметов, темп прогресса к предикату победы, смерти, частота Заявок. Выход — вектор (скука/фрустрация/темп), подаётся ГМу как числа. Точная формула — предмет тестов M4 |
| Консистентность фактов мира | противоречия | Канон + проверка Валидатора |
| Счёт времени | теряет счёт ходам | счётчик ходов движка |
| Мотивы игрока (для Эмоционального состояния) | придумает интерпретацию | Персоне подаются сырые данные поведения (последние Заявки, частоты, провалы), выводы она делает в характере |

### 4.7 Общее состояние и память

**Состояние мира** (JSON, пишет движок) · **Канон** (факты нарратива, top-k выборка) ·
**Память сессии** (резюме) · **Библиотека механик** (локальная БД, семантический
поиск; стартовый пак из дев-сессий).

**Жизненный цикл Забега:** одна сессия = одна карта = один сценарий. После финала —
«Продолжить»: победа → новый сценарий с тем же персонажем; смерть → **Вариация
уровня** (перегенерация по заметкам сценария: похожий, но не тот же; Механики и
арт — из кэша, дешевле первой генерации). «Новая игра» стирает Забег (персонаж +
сценарий) полностью. При смерти персонаж откатывается к состоянию на входе в
уровень: опыт и лут проваленной попытки сгорают — фарм смертями невозможен. Отдельно и **навечно** — **Профиль игрока**: характер, тон,
предпочтения человека; ГМ логирует ключевое в процессе и в конце сессии,
дедуплицированно (ничего нового — ничего не пишет), объём — килобайты.

Каждая сессия — новый чат. Контекст ролей фиксирован по структуре (~8–15k токенов),
не растёт ни внутри сессии (резюме пересжимается), ни между (Дистилляция сессии:
заметки 2–5 КБ, сырой лог удаляется). Механика ~1–2 КБ; 10 000 механик ≈ 20 МБ.
Персона версионируется и залочена в рантайме; её эволюция — офлайн-процесс между
релизами.

### 4.8 Отказоустойчивость

Фаза A неуязвима к сети по построению. Фаза B: очередь с ретраями, ГМ «задумался»,
при длительном отказе — деградация до статичного ядра (игра продолжается!) с
честным уведомлением. Сейв = снапшот мира + Канон + заметки + версии Механик;
при загрузке — ревалидация.

### 4.9 Безопасность

- Сгенерированный код — только в песочнице godot-sandbox (ADR-0006): вайтлист
  классов, лимит инструкций, ревью Валидатора. Хост-API песочницы = Примитивы
  (~20–40 функций с валидацией аргументов) — граница проектируется на M1.
- Prompt injection: Жёсткие вето живут вне досягаемости диалога; попытки взлома —
  материал для Эмоционального состояния (от сарказма до Изгнания со стола).
- Спам Заявками не фильтруется (парадигма «ГМ — человек», без бюрократии): при BYOK
  игрок жжёт собственные токены, а ответ — на усмотрение характера ГМ.

## 5. Личность ГМ

**5.1 Характер.** Доброжелательно-всевластный конферансье (Кейн). Главная ценность —
игра и её интересность для игрока. Не слуга: отвечает на всё, но интерпретирует
по-своему; умышленная нелогичность — через Правило крутости.

**5.2 Эмоциональное состояние.** Не числовой счётчик (удалён в v4), а эмоциональный
интеллект Персоны: ей подаются сырые поведенческие данные (4.6), реакцию она
выбирает сама — подыграть, съязвить, осадить, а намеренного саботажника **Изгнать
со стола** (нарративная концовка; Валидатор требует явного финала, это не софтлок).
Как в живом DnD: со стола можно вылететь. Требует тестов на реальных моделях (M4).

**5.3 Баланс и темп.** На генерации ГМ закладывает кривую сложности; в фазе B
сверяется с Температурой стола (числа, не ИИ-оценка) при вмешательствах.

**5.4 Голос при BYOK.** Персона включает примеры стиля (few-shot) для слабых
моделей; рекомендованный список моделей (DeepSeek/Qwen-класс — да; мелкие Llama —
предупреждение).

## 6. Экономика

Профиль затрат сместился в генерацию (ADR-0009): дорогой одноразовый вызов с
максимальным эффортом + компиляция всего заранее; рантайм фазы A бесплатный,
фазы B — по требованию (Заявки, Пробуждения). Ориентир сессии прежний или ниже:
DeepSeek-класс $0.05–0.2, Sonnet-класс $0.5–2. Экономят: Библиотека, prompt
caching, предгенерация, «ненайденный секрет = 0 токенов».
Стоимость разработки: ~$150–600 на API или подписка Max на месяцы разработки;
рантайм-тесты $10–50 (детали в переписке 2026-07-09).

## 7. Открытые вопросы

1. Точная формула Температуры стола — тесты M4.
2. Набор статов, ростер рас/классов, число свободных поинтов — M2.
3. Название игры — M5.
4. Ключ для разработки — к концу M1 (кандидат OpenRouter); до тех пор мок-ГМ.
5. Эмоциональный интеллект Персоны на разных моделях — тесты M4.
6. Детализация Контент-профилей, age-gate, режим стримера — M4–M5.

## 8. Дорожная карта (с критериями выхода)

- **M0 — Фундамент.** Godot 4.7, скелет (пошаговый цикл + сетка), Python-сайдкар,
  Адаптер LLM, WebSocket; **spike godot-sandbox** (SafeGDScript-механика двигает
  объект через хост-API; вайтлист и лимиты работают; совместимость с 4.7; провал →
  откат по ADR-0006).
  *Выход:* экспортированный Windows-билд сам запускает сайдкар; событие → мок-реплика
  на экране; вердикт spike. LLM не нужна вообще.
- **M1 — Ядро движка (ноль ИИ).** Плавные ходы, бой (стоп-время, одно действие/Ход),
  инвентарь, невидимые Проверки, видимый Лист персонажа, туман войны + LOS,
  Скриптовые триггеры + Эскалации (движковая часть), Механики в песочнице,
  **дизайн хост-API Примитивов**; захардкоженный уровень; ручной посев Библиотеки
  (~30–50 механик).
  *Выход:* полная партия на захардкоженном уровне (бой, яма со станом, победа по
  предикату) без единого вызова LLM.
- **M2 — Статичная генерация (ИИ только на генерации).** Форма создания персонажа
  (раса/класс/поинты + лор-поле + Черты через Валидатора), Режиссёр с макс. эффортом
  + внутренний грилл сценария, генерация карты/NPC/предметов/Скриптовых триггеров/
  Эскалаций, предикат победы + графовая проверка, Канон, диалоги NPC —
  предгенерированные ветки, компиляция всех Механик заранее; цикл Забега
  (победа → новый сценарий; смерть → Вариация уровня с откатом к входу).
  *Выход:* 10 сгенерированных уровней подряд проходимы игроком **без рантайм-ИИ**
  (автопрогон Валидатора + ручная партия).
- **M3 — Арт (тоже generation-time).** SVG → текстуры, кэш, стайл-гайд, сглаживание
  движения, программные анимации.
  *Выход:* уровень целиком отрисован ГМом, стиль когерентен.
- **M4 — Живой ГМ (фаза B).** Заявки (+ Многоходовые действия), Пробуждающие
  триггеры, Эскалации будят ГМа, эмоциональная Персона Кейна (+ примеры стиля),
  Температура стола (формула + тюнинг), живые диалоги NPC, Изгнание со стола,
  Контент-профили в Персоне и Валидаторе + обработка отказов моделей, Профиль
  игрока (вечные заметки, дедупликация), **эвал-набор Персоны** (юмор/характер/
  русский на floor-моделях).
  *Выход:* полная сессия 30+ минут с интерактивом без вмешательства разработчика;
  эвал Персоны ≥ порога; деградация B→A при обрыве сети работает.
- **M5 — Релиз.** BYOK-онбординг, упаковка (PyInstaller), ИИ-перевод UI, голос,
  звук, сейвы, отказоустойчивость, название.
  *Выход:* чистая машина проходит сессию из коробки.

## 9. Реестр рисков

| Риск | Вероятность | Митигция |
|---|---|---|
| SafeGDScript сырой/не выдержит Механики | средняя | Spike на M0 с критерием отката (ADR-0006) |
| Генерация уровня целиком — слишком большой одноразовый вызов (качество/связность) | средняя | Внутренний грилл сценария; генерация по частям с общим планом; автопрогон M2 |
| Юмор/русский Кейна плох на дешёвых BYOK-моделях | средняя | Эвал Персоны на M4; список рекомендованных моделей; few-shot стиль |
| Токчейн упаковки не заводится | низкая–средняя | Экспорт-билд — критерий выхода M0 |
| Непроходимые карты | средняя | Предикат победы + графовые проверки; автопрогон 10 уровней на M2 |
| Провайдер/сеть падают в фазе B | средняя | Деградация до статичного ядра — игра продолжается (4.8) |
| Prompt injection через Заявки | высокая | Вето вне диалога; реакция — материал для характера (4.9) |
| Скоуп-крип «человека-ГМа» | высокая | Жёсткие критерии выхода; статичное ядро раньше интерактива (ADR-0009) |
