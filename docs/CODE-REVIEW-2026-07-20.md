# Ревизия кода (2026-07-20) — карта должна быть данными

Повод: захардкоженная геометрия уровня в движковом скрипте (`world_grid.gd`),
всплывшая через B-006. Пользователь потребовал полную ревизию.

## Главная проблема

`world_grid.gd` (представление) держит **данные уровня**: `GRID_W/GRID_H`,
`PIT_CELL`, `is_wall()` с `DIVIDER_X/DOORWAY_Y`. Нарушает ADR-0013 (World State —
единственный источник правды) и цель проекта: Режиссёр не сможет сгенерировать
карту, если геометрия вычисляется в коде движка. B-006 — прямое следствие
(яма захардкожена в `world_grid` И `level.gd` с разными координатами).

Вторично: `primitives.gd` и `player.gd` берут проходимость из `world_grid`
(вьюха), а не из данных → логика зависит от представления.

## Что чисто

- Координаты в `level.gd` — корректно: уровень объявляет контент как данные
  (на M2 их генерит Режиссёр).
- UI-константы (позиции/размеры панелей) — норма.

## План рефакторинга

1. **World State (`world_state.gd`)**: тайл-карта — `map_w/map_h` + `_tiles`
   (0=пол, 1=стена), методы `set_map/in_bounds/is_wall/is_walkable`, в JSON.
2. **Primitives**: `is_walkable` из World, убрать зависимость от grid-ноды;
   конструктор без grid.
3. **`world_grid.gd`**: чистый рендерер тайлов из World + туман. Ноль контента.
4. **`level.gd`**: строит карту как данные (границы+стена+проём) → `World.set_map`.
5. **Яма**: рендер из World-объекта в `entity_renderer` (revealed prop). Убирает B-006.

## После фундамента (на чистом коде)

Диагональ игроку (не отбирать у NPC), game-over через `get_tree().paused` +
оверлей (PROCESS_MODE_ALWAYS) + `reload_current_scene()`, падение врага в яму,
скролл-лог dev-HUD, UI (HP/статы всегда, инвентарь: HUD-слоты 1-5 + рюкзак-драг),
камера за игроком, окно больше. Нарратор-прерывание новой репликой — уже сделано.
