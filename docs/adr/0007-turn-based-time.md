# Вся игра пошаговая (модель NetHack/Stoneshard), а не реалтайм

Рассматривались: реалтайм (первоначальная рекомендация), пошаговый бой при
реалтаймовом мире, реалтайм с тактической паузой. Решение пользователя
(2026-07-09): пошаговость везде. Вне боя — Плавные ходы (время идёт, пока игрок
действует, пауз не видно); в бою время останавливается, у игрока одно действие
за Ход: базовое (удар, зелье) или Заявка ГМу; сложные Заявки становятся
Многоходовыми действиями (длительность назначает ГМ по реалистичности).

Почему это лучше реалтайма для этой игры: (1) каждый Ход — дискретное
семантическое событие, Наблюдателю не нужен непрерывный поток; (2) латентность
LLM прячется в структуру ходов («ход противника»), а не только в паузы
рассказчика; (3) Заявка — родная петля настольного DnD; (4) стоимость в ходах —
естественный балансир произвольных идей игрока. Следствия: одна боевая система;
исчезает требование непрерывной физики; движение и действия учитываются ходами.
