# Примитив = функция, Механика = композиция-данные, Библиотека = Примитивы+Механики

Уточняет ADR-0004 после грилла 2026-07-18 (пользователь поправил терминологию,
которую я путал: сгенерированный код называл «Механикой»). Жёстко фиксируем:

- **Примитив** — функция, единица исполняемого поведения (`set_prop`, `spawn`,
  `damage`, `roll_check`...). Два происхождения:
  - **ванильные** — написаны нами, едут в билде, безопасный хост-API
    ([game/scripts/primitives.gd](../../game/scripts/primitives.gd));
  - **сгенерированные** — Создатель пишет в рантайме как **сырой код**, когда
    существующих не хватает; исполняются в песочнице, где доступны только другие
    Примитивы. Это ЕДИНСТВЕННЫЙ сырой код в игре (→ ADR-0006).
- **Механика** — **композиция вызовов Примитивов** (рецепт-данные: триггер + шаги),
  НЕ сырой код. Крутится нашим доверенным интерпретатором. Всегда данные →
  безопасна и непереносима как код.
- **Библиотека** — хранилище Примитивов + Механик (Механика без своих Примитивов не
  работает). Две раздельные: **Ванильная** (в билде) и **игрока** (локальная, растёт
  за сессии, не удаляется).

Поток: игрок просит уникальное → нет подходящего Примитива → **Создатель** пишет
новый Примитив (сырой код, песочница) + Механику; всё падает в Библиотеку игрока.
Позже игрок просит похожее → **Сборщик** (дёшево) собирает Механику из тех же
Примитивов, Создатель не нужен. Так дорогая роль вызывается всё реже.

Следствия: сырой код изолирован в одном месте (сгенерированный Примитив в песочнице)
— вся остальная генерация это безопасные данные-композиции; терминология в коде и
доках должна следовать этому строго (Механику сырым кодом не называть).
