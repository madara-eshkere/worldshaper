# Энергетический планировщик ходов (независимые актёры), а не цикл вокруг игрока

Первая пошаговая реализация была построена ВОКРУГ игрока: игрок делает действие →
все мобы ходят по разу (до/после игрока по скорости). Это дало класс багов
(2026-07-20): при побеге зазор до быстрого моба оставался постоянным (скорость не
давала лишних ходов), фазы «до/после» плодили неровное число атак, тайминг фаз
пускал игрока и моба в одну клетку.

Решение — **энергетический планировщик** ([turn_scheduler.gd](../../game/scripts/turn_scheduler.gd)),
стандартный для роглайков (NetHack/Stoneshard). У каждого **актёра** (игрок + живые
враги) своё время следующего хода `next_at`; ход получает актёр с минимальным
`next_at`; после хода `next_at += 1.0 / speed`. Быстрый (speed 8) приходит вдвое чаще
speed-4 и реально догоняет. Игрок — обычный актёр: планировщик крутит ходы мобов,
пока не подойдёт очередь игрока, затем ждёт ввод; игрок вызывает `player_acted()`
после каждого своего действия. AI мобов (шаг к игроку/атака, занятость клетки,
ямы, полёт) живёт в планировщике; отдельный `enemy_controller` удалён.

Следствия: скорость = частота хода (никаких «клеток за ход»-костылей); строгая
занятость — актёр не встаёт в клетку другого (атакует из соседней); «важность» и
Эскалации (ADR-0009) по-прежнему в Рефлексах, планировщик отвечает только за
порядок ходов боя. Это фундамент, на который ИИ будет вешать своих существ и
механики — поэтому оформлено как ADR.
