# Экономика: инференс оплачивает разработчик; модели — сменные через Адаптер

Пересматривает ADR-0005 (там релиз v1 был BYOK — игрок приносит ключ). Решение
пользователя (2026-07-17): для этой игры нужна КОНКРЕТНАЯ выверенная модель со
специальным, протестированным разработчиком промптом — качество контента должно
быть гарантировано. Поэтому:

1. **Отказ от «модель игрока = модель ГМ».** Возможность подключать свою локальную
   модель НЕ вводим (пока). Игрок не выбирает модель.
2. **OpenRouter — единая платформа подключения** (все модели за одним API); под неё
   пишется логика Адаптера LLM.
3. **Платит разработчик.** Аккаунт у разработчика; игроки платят подпиской или в
   цене игры. Следствие: на релизе нужен **наш бэкенд-прокси** (игра → наш сервер с
   нашим ключом → OpenRouter), авторизация игрока, рейт-лимиты, защита от абьюза,
   мониторинг расходов, биллинг. «Ноль инфраструктуры» из ADR-0005 отменяется.
4. **Каждый токен — наши деньги** → экономия (статичное ядро, редкие Чекпоинты,
   предзаписанные Реплики, Сборщик-первым, Библиотека) становится условием
   выживания, а выбор модели на роль = прямо наш счёт.
5. **Модель — строка конфига (Адаптер LLM).** Конкретные модели на роли НЕ
   фиксируются заранее: тестируются вручную (эвал Персоны на M4, от Opus 4.8 вниз до
   DeepSeek/Qwen), берётся самая дешёвая, прошедшая порог качества (юмор, русский,
   логика, галлюцинации). Ростер ролей в DESIGN.md — ориентир, не обязательство.
6. **Dev-путь:** в разработке — прямые вызовы Claude через личный ключ пользователя
   (Anthropic-direct бэкенд в Адаптере, рядом с OpenRouter). На первом релизе Claude
   скорее всего НЕ используется — слишком дорого для developer-paid.
