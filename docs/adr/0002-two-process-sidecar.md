# Два процесса: игра (Godot) + ИИ-сайдкар (Python) через WebSocket

Мультимодельная оркестрация (роли, tool-loops, ретраи, стриминг, семантический поиск)
на GDScript мучительна и не имеет экосистемы. Решение: игра шлёт семантические
события в локальный Python-сайдкар, тот возвращает Намерения/реплики; игра запускает
сайдкар сама. Альтернатива «всё в GDScript через HTTPRequest» отклонена: нет SDK,
нет нормальных embeddings/оркестрации. Следствие: релизная сборка должна паковать
сайдкар (PyInstaller onefile) — это принятый риск упаковки.
