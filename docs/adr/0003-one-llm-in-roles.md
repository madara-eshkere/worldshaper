# Одна сильная LLM в ролях, а не свои обученные сети

Рассматривалось обучение узких нейросетей под каждую задачу ГМ. Отклонено: задачи ГМ
(юмор, сценарий, интерпретация намерений) — открытые языковые задачи, узкие сети их
не решают, а обучение/файнтюн своей LLM нереалистичен по данным и стоимости; при этом
требование «минимум галлюцинаций» достигается заземлением (Канон, Состояние мира,
вето), а не заменой модели. Решение: роли Режиссёр/Наблюдатель/Инженер/Логик/Валидатор —
разные контексты одной-двух базовых моделей, доступ через Адаптер LLM
(OpenAI-совместимый; модель = конфигурация). «Обучение» ГМ = память + дистилляция
сессий + Библиотека механик + офлайн-эволюция Персоны (версионируется и лочится).
Дообучение (дистилляция на логах реальных сессий) — допустимая оптимизация фазы v2+.
