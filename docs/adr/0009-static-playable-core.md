# Статичное ядро: игра проходима вообще без рантайм-ИИ; наблюдатель-LLM удалён

Рассматривалась архитектура с LLM-Наблюдателем, фильтрующим поток событий в
рантайме. Отклонена (решение пользователя, 2026-07-09): наблюдатель-LLM — это
запрос-ответ на каждый ход (лишняя задержка и стоимость), а решение «что важно»
лучше всего принимает сам Режиссёр в момент генерации уровня — у него максимум
контекста, он автор всех секретов.

Решение — две фазы рантайма:

1. **Статичное ядро (фазы M1–M3).** Весь ИИ отрабатывает на генерации: Режиссёр с
   максимальным эффортом (+ внутренний грилл сценария Логиком/Валидатором) пишет
   сценарий, карту, NPC, предметы, Механики, Скриптовые триггеры с реакциями,
   Эскалации и предикат победы; Инженер всё компилирует заранее. В рантайме игра
   полностью проходима с **нулём вызовов LLM**.
2. **Живой ГМ (M4+).** Поверх статичного ядра включаются: Заявки игрока,
   Пробуждающие триггеры, Эскалации (детерминированные счётчики превращают
   Скриптовый триггер в Пробуждающий: «10 проваленных проверок подряд — вмешайся»),
   эмоциональная Персона.

Роль наблюдения переходит к детерминированным метрикам в Рефлексах (счётчики
неудач/стагнации, Температура стола по формуле) — надёжным не-ИИ инструментам,
которые дают ГМу контекст, а не суждения. Следствия: надёжность и стоимость
рантайма перестают зависеть от LLM; секрет, который игрок не нашёл, не стоит ни
токена; динамизм по умолчанию ниже — возвращается на M4 как слой поверх.
